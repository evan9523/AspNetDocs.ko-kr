### <a name="custom-shaped-viewmodel-classes"></a><span data-ttu-id="a9e95-156">사용자 지정 모양의 ViewModel 클래스</span><span class="sxs-lookup"><span data-stu-id="a9e95-156">Custom-shaped ViewModel classes</span></span>

<span data-ttu-id="a9e95-157">위의 시나리오 DinnerFormViewModel 클래스 직접 지원 SelectList 모델 속성과 함께 속성으로 Dinner 모델 개체를 노출합니다.</span><span class="sxs-lookup"><span data-stu-id="a9e95-157">In the scenario above, our DinnerFormViewModel class directly exposes the Dinner model object as a property, along with a supporting SelectList model property.</span></span> <span data-ttu-id="a9e95-158">이 방법은 시나리오는 우리의 보기 템플릿에서 생성 하려는 HTML UI 비교적 밀접 하 게에 해당 도메인 모델 개체에 대 한 제대로 작동 합니다.</span><span class="sxs-lookup"><span data-stu-id="a9e95-158">This approach works fine for scenarios where the HTML UI we want to create within our view template corresponds relatively closely to our domain model objects.</span></span>

<span data-ttu-id="a9e95-159">이러한 방식이 대 한 경우 사용할 수 있는 한 가지 옵션을 사용자 지정 모양의 ViewModel 클래스 뷰에서 – 소비 해당 개체 모델은 더 최적화 하는 기본 도메인 모델 개체와 완전히 다르게 보일 수 있습니다를 만드는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="a9e95-159">For scenarios where this isn't the case, one option that you can use is to create a custom-shaped ViewModel class whose object model is more optimized for consumption by the view – and which might look completely different from the underlying domain model object.</span></span> <span data-ttu-id="a9e95-160">예를 들어, 다른 속성 이름 및/또는 여러 모델 개체에서 수집 된 집계 속성 노출 될 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a9e95-160">For example, it could potentially expose different property names and/or aggregate properties collected from multiple model objects.</span></span>
