## <a name="aspnet-web-forms-helps-you-overcome-challenges"></a><span data-ttu-id="28292-130">ASP.NET Web Forms를 사용 하면 문제를 해결할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-130">ASP.NET Web Forms Helps You Overcome Challenges</span></span>

<span data-ttu-id="28292-131">웹 응용 프로그램 프로그래밍은 일반적인 클라이언트 기반 응용 프로그램을 프로그래밍할 때 일반적으로 발생 하지 않는 문제를 제공 합니다.</span><span class="sxs-lookup"><span data-stu-id="28292-131">Web application programming presents challenges that do not typically arise when programming traditional client-based applications.</span></span> <span data-ttu-id="28292-132">문제는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-132">Among the challenges are:</span></span>

- <span data-ttu-id="28292-133">**풍부한 웹 사용자 인터페이스 구현** -특히 페이지에 복잡 한 레이아웃, 많은 양의 동적 콘텐츠, 완전 한 기능을 갖춘 사용자 대화형 개체가 있는 경우에는 기본적인 HTML 기능을 사용 하 여 사용자 인터페이스를 디자인 하 고 구현 하는 것이 어려울 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-133">**Implementing a rich Web user interface** - It can be difficult and tedious to design and implement a user interface using basic HTML facilities, especially if the page has a complex layout, a large amount of dynamic content, and full-featured user-interactive objects.</span></span>
- <span data-ttu-id="28292-134">**클라이언트와 서버를 분리** 하는 웹 응용 프로그램, 클라이언트 (브라우저) 및 서버는 서로 다른 컴퓨터 (및 다른 운영 체제 에서도)에서 실행 되는 다른 프로그램입니다.</span><span class="sxs-lookup"><span data-stu-id="28292-134">**Separation of client and server** - In a Web application, the client (browser) and server are different programs often running on different computers (and even on different operating systems).</span></span> <span data-ttu-id="28292-135">따라서 응용 프로그램의 두 절반은 거의 정보를 공유 하지 않습니다. 이러한 사용자는 통신할 수 있지만 일반적으로 간단한 정보의 작은 청크만 교환 합니다.</span><span class="sxs-lookup"><span data-stu-id="28292-135">Consequently, the two halves of the application share very little information; they can communicate, but typically exchange only small chunks of simple information.</span></span>
- <span data-ttu-id="28292-136">**상태 비저장 실행** -웹 서버에서 페이지에 대 한 요청을 받으면 페이지를 찾아 처리 하 고 브라우저에 보낸 다음 모든 페이지 정보를 삭제 합니다.</span><span class="sxs-lookup"><span data-stu-id="28292-136">**Stateless execution** - When a Web server receives a request for a page, it finds the page, processes it, sends it to the browser, and then discards all page information.</span></span> <span data-ttu-id="28292-137">사용자가 같은 페이지를 다시 요청 하면 서버에서 전체 시퀀스를 반복 하 여 페이지를 처음부터 다시 처리 합니다.</span><span class="sxs-lookup"><span data-stu-id="28292-137">If the user requests the same page again, the server repeats the entire sequence, reprocessing the page from scratch.</span></span> <span data-ttu-id="28292-138">다른 방법으로 서버에 처리 된 페이지의 메모리가 없는 경우-페이지는 상태 비저장입니다.</span><span class="sxs-lookup"><span data-stu-id="28292-138">Put another way, a server has no memory of pages that it has processed—page are stateless.</span></span> <span data-ttu-id="28292-139">따라서 응용 프로그램이 페이지에 대 한 정보를 유지 관리 해야 하는 경우 상태 비저장 특성으로 인해 문제가 될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-139">Therefore, if an application needs to maintain information about a page, its stateless nature can become a problem.</span></span>
- <span data-ttu-id="28292-140">**알 수 없는 클라이언트 기능** -대부분의 경우 다양 한 브라우저를 사용 하 여 많은 사용자가 웹 응용 프로그램에 액세스할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-140">**Unknown client capabilities** - In many cases, Web applications are accessible to many users using different browsers.</span></span> <span data-ttu-id="28292-141">브라우저는 다양 한 기능을 제공 하므로 모든 기능에서 동일 하 게 실행 되는 응용 프로그램을 만들기 어렵습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-141">Browsers have different capabilities, making it difficult to create an application that will run equally well on all of them.</span></span>
- <span data-ttu-id="28292-142">기존 웹 응용 프로그램에서 데이터 **에 대 한** 읽기 및 쓰기는 복잡 하 고 리소스를 많이 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="28292-142">**Complications with data access** - Reading from and writing to a data source in traditional Web applications can be complicated and resource-intensive.</span></span>
