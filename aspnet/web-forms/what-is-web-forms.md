# <a name="what-is-web-forms"></a>Web Forms 정의

ASP.NET Web Forms는 ASP.NET 웹 응용 프로그램 프레임 워크의 일부 이며 [Visual Studio](https://www.asp.net/downloads)에 포함 되어 있습니다. ASP.NET 웹 응용 프로그램을 만드는 데 사용할 수 있는 네 가지 프로그래밍 모델 중 하나 이며, 다른 응용 프로그램은 ASP.NET MVC, ASP.NET 웹 페이지 및 ASP.NET 단일 페이지 응용 프로그램입니다.

Web Forms은 사용자가 브라우저를 사용 하 여 요청 하는 페이지입니다. 이러한 페이지는 HTML, 클라이언트 스크립트, 서버 컨트롤 및 서버 코드의 조합을 사용 하 여 작성할 수 있습니다. 사용자가 페이지를 요청 하면 프레임 워크가 서버에서 컴파일되고 실행 된 다음 프레임 워크는 브라우저에서 렌더링할 수 있는 HTML 태그를 생성 합니다. ASP.NET Web Forms 페이지는 브라우저 또는 클라이언트 장치에서 사용자에 게 정보를 제공 합니다.

Visual Studio를 사용 하 여 ASP.NET Web Forms를 만들 수 있습니다. Visual Studio IDE (통합 개발 환경)를 사용 하면 서버 컨트롤을 끌어서 놓아 Web Forms 페이지의 레이아웃을 만들 수 있습니다. 그런 다음 페이지 또는 페이지의 컨트롤에 대 한 속성, 메서드 및 이벤트를 쉽게 설정할 수 있습니다. 이러한 속성, 메서드 및 이벤트는 웹 페이지의 동작, 모양 및 느낌을 정의 하는 데 사용 됩니다. 페이지의 논리를 처리 하는 서버 코드를 작성 하려면 Visual Basic 또는 C#와 같은 .net 언어를 사용할 수 있습니다.

> [!NOTE] 
> 
> ASP.NET 및 Visual Studio 설명서는 여러 버전으로 확장 됩니다. 이전 버전의 기능을 강조 하는 항목은 최신 버전을 사용 하는 현재 작업과 시나리오에 유용할 수 있습니다.

**ASP.NET Web Forms는 다음과 같습니다.** 

- Microsoft ASP.NET 기술에 기반 하 여 서버에서 실행 되는 코드는 브라우저 또는 클라이언트 장치에 웹 페이지 출력을 동적으로 생성 합니다.
- 모든 브라우저 또는 모바일 장치와 호환 됩니다. ASP.NET 웹 페이지는 스타일, 레이아웃 등과 같은 기능에 대 한 올바른 브라우저 호환 HTML을 자동으로 렌더링 합니다.
- Microsoft Visual Basic 및 Microsoft Visual C#과 같은 .net 공용 언어 런타임에서 지 원하는 모든 언어와 호환 됩니다.
- Microsoft .NET 프레임 워크를 기반으로 합니다. 이는 관리 되는 환경, 형식 안전성 및 상속을 포함 하 여 프레임 워크의 모든 이점을 제공 합니다.
- 사용자가 만든 및 타사 컨트롤을 해당 사용자에 게 추가할 수 있으므로 유연 합니다.

**ASP.NET Web Forms 제안:** 

- HTML 및 기타 UI 코드를 응용 프로그램 논리와 분리 합니다.
- 데이터 액세스를 비롯 한 일반적인 작업을 위한 다양 한 서버 컨트롤입니다.
- 뛰어난 도구 지원을 갖춘 강력한 데이터 바인딩.
- 브라우저에서 실행 되는 클라이언트 쪽 스크립팅을 지원 합니다.
- 라우팅, 보안, 성능, 국제화, 테스트, 디버깅, 오류 처리 및 상태 관리를 비롯 한 다양 한 기타 기능을 지원 합니다.

## <a name="aspnet-web-forms-helps-you-overcome-challenges"></a>ASP.NET Web Forms를 사용 하면 문제를 해결할 수 있습니다.

웹 응용 프로그램 프로그래밍은 일반적인 클라이언트 기반 응용 프로그램을 프로그래밍할 때 일반적으로 발생 하지 않는 문제를 제공 합니다. 문제는 다음과 같습니다.

- **풍부한 웹 사용자 인터페이스 구현** -특히 페이지에 복잡 한 레이아웃, 많은 양의 동적 콘텐츠, 완전 한 기능을 갖춘 사용자 대화형 개체가 있는 경우에는 기본적인 HTML 기능을 사용 하 여 사용자 인터페이스를 디자인 하 고 구현 하는 것이 어려울 수 있습니다.
- **클라이언트와 서버를 분리** 하는 웹 응용 프로그램, 클라이언트 (브라우저) 및 서버는 서로 다른 컴퓨터 (및 다른 운영 체제 에서도)에서 실행 되는 다른 프로그램입니다. 따라서 응용 프로그램의 두 절반은 거의 정보를 공유 하지 않습니다. 이러한 사용자는 통신할 수 있지만 일반적으로 간단한 정보의 작은 청크만 교환 합니다.
- **상태 비저장 실행** -웹 서버에서 페이지에 대 한 요청을 받으면 페이지를 찾아 처리 하 고 브라우저에 보낸 다음 모든 페이지 정보를 삭제 합니다. 사용자가 같은 페이지를 다시 요청 하면 서버에서 전체 시퀀스를 반복 하 여 페이지를 처음부터 다시 처리 합니다. 다른 방법으로 서버에 처리 된 페이지의 메모리가 없는 경우-페이지는 상태 비저장입니다. 따라서 응용 프로그램이 페이지에 대 한 정보를 유지 관리 해야 하는 경우 상태 비저장 특성으로 인해 문제가 될 수 있습니다.
- **알 수 없는 클라이언트 기능** -대부분의 경우 다양 한 브라우저를 사용 하 여 많은 사용자가 웹 응용 프로그램에 액세스할 수 있습니다. 브라우저는 다양 한 기능을 제공 하므로 모든 기능에서 동일 하 게 실행 되는 응용 프로그램을 만들기 어렵습니다.
- 기존 웹 응용 프로그램에서 데이터 **에 대 한** 읽기 및 쓰기는 복잡 하 고 리소스를 많이 사용할 수 있습니다.
- **확장성과 관련 된** 문제-대부분의 경우 기존 메서드로 디자인 된 웹 응용 프로그램은 응용 프로그램의 다양 한 구성 요소 간에 호환성이 부족 하기 때문에 확장성 목표를 충족 하지 못합니다. 자주 발생 하는 응용 프로그램에 대 한 일반적인 오류 지점입니다.

웹 응용 프로그램에 대해 이러한 문제를 충족 하려면 상당한 시간과 노력이 필요할 수 있습니다. ASP.NET Web Forms 및 ASP.NET 프레임 워크는 다음과 같은 방법으로 이러한 문제를 해결 합니다.

- **직관적이 고 일관적인 개체 모델** -ASP.NET 페이지 프레임 워크는 폼을 별도의 클라이언트 및 서버 조각이 아닌 단위로 간주할 수 있는 개체 모델을 제공 합니다. 이 모델에서는 페이지 요소에 대 한 속성을 설정 하 고 이벤트에 응답 하는 기능을 포함 하 여 기존 웹 응용 프로그램 보다 직관적인 방법으로 페이지를 프로그래밍할 수 있습니다. 또한 ASP.NET 서버 컨트롤은 HTML 페이지의 물리적 콘텐츠와 브라우저와 서버 간의 직접 상호 작용에서 추상화 됩니다. 일반적으로 클라이언트 응용 프로그램에서 컨트롤을 사용할 수 있는 방식으로 서버 컨트롤을 사용할 수 있으며, 컨트롤 및 해당 내용을 제공 하 고 처리 하는 HTML을 만드는 방법을 고려할 필요가 없습니다.
- **이벤트 구동 프로그래밍 모델** -ASP.NET는 클라이언트 또는 서버에서 발생 하는 이벤트에 대 한 이벤트 처리기를 작성 하는 친숙 한 모델을 웹 응용 프로그램으로 가져올 Web Forms. ASP.NET 페이지 프레임 워크는 클라이언트에서 이벤트를 캡처하고 서버에 전송 하 고 적절 한 메서드를 호출 하는 기본 메커니즘을 모두 자동 및 보이지 않는 방식으로이 모델을 추상화 합니다. 결과적으로 이벤트 기반 개발을 지 원하는 명확 하 고 작성 된 코드 구조가 생성 됩니다.
- **직관적인 상태 관리** -ASP.NET 페이지 프레임 워크는 페이지 및 해당 컨트롤의 상태를 유지 관리 하는 작업을 자동으로 처리 하 고, 응용 프로그램 관련 정보의 상태를 유지 관리 하는 명시적인 방법을 제공 합니다. 이는 서버 리소스를 과도 하 게 사용 하지 않고 수행 되며 쿠키를 브라우저에 전송 하거나 사용 하지 않고 구현할 수 있습니다.
- **브라우저 독립적 응용 프로그램** -ASP.NET 페이지 프레임 워크를 사용 하면 서버에 모든 응용 프로그램 논리를 만들 수 있으므로 브라우저의 차이점을 명시적으로 코딩할 필요가 없습니다. 그러나이 기능을 사용 하면 클라이언트 쪽 코드를 작성 하 여 향상 된 성능과 풍부한 클라이언트 환경을 제공 함으로써 브라우저 관련 기능을 활용할 수 있습니다.
- **.NET Framework 공용 언어 런타임 지원** -ASP.NET 페이지 프레임 워크는 .NET Framework를 기반으로 하므로 모든 ASP.NET 응용 프로그램에서 전체 프레임 워크를 사용할 수 있습니다. 응용 프로그램은 런타임과 호환 되는 모든 언어로 작성할 수 있습니다. 또한 ADO.NET를 포함 하 여 .NET Framework에서 제공 하는 데이터 액세스 인프라를 사용 하 여 데이터 액세스를 단순화 합니다.
- **.NET Framework 확장 가능한 서버 성능** -ASP.NET 페이지 프레임 워크를 사용 하면 응용 프로그램 논리를 복잡 하 게 변경 하지 않고 단일 프로세서를 사용 하는 단일 컴퓨터에서 다중 컴퓨터 웹 팜으로 완전히 확장 하 여 웹 응용 프로그램을 확장할 수 있습니다.

## <a name="features-of-aspnet-web-forms"></a>ASP.NET Web Forms의 기능

- **서버 컨트롤**-ASP.NET 웹 서버 컨트롤은 페이지가 요청 될 때 실행 되 고 브라우저에 태그를 렌더링 하는 ASP.NET 웹 페이지의 개체입니다. 많은 웹 서버 컨트롤은 단추 및 텍스트 상자와 같은 익숙한 HTML 요소와 비슷합니다. 다른 컨트롤은 일정 컨트롤과 같은 복잡 한 동작을 포함 하 고 데이터 원본에 연결 하 고 데이터를 표시 하는 데 사용할 수 있는 컨트롤을 포함 합니다.
- **마스터 페이지**-ASP.NET 마스터 페이지를 사용 하면 응용 프로그램의 페이지에 대해 일관 된 레이아웃을 만들 수 있습니다. 단일 마스터 페이지는 애플리케이션의 모든 페이지(또는 페이지 그룹)에 대해 원하는 모양과 느낌 및 표준 동작을 정의합니다. 그런 다음 표시할 콘텐츠를 포함하는 개별 콘텐츠 페이지를 만들 수 있습니다. 사용자가 콘텐츠 페이지를 요청 하면 마스터 페이지와 병합 하 여 마스터 페이지의 레이아웃과 콘텐츠 페이지의 콘텐츠를 결합 하는 출력을 생성 합니다.
- **데이터 ASP.NET 작업**은 데이터를 저장, 검색 및 표시 하는 다양 한 옵션을 제공 합니다. ASP.NET Web Forms 응용 프로그램에서는 데이터 바인딩된 컨트롤을 사용 하 여 테이블, 텍스트 상자 및 드롭다운 목록과 같은 웹 페이지 UI 요소에서 데이터의 표시 또는 입력을 자동화할 수 있습니다.
- **멤버 자격**-ASP.NET Identity는 응용 프로그램에서 만든 데이터베이스에 사용자의 자격 증명을 저장 합니다. 사용자가 로그인 하면 응용 프로그램은 데이터베이스를 읽어 자격 증명의 유효성을 검사 합니다. 프로젝트의 *계정* 폴더에는 등록, 로그인, 암호 변경, 액세스 권한 부여의 여러 부분을 구현 하는 파일이 포함 되어 있습니다. 또한 ASP.NET Web Forms는 OAuth 및 Openid connect를 지원 합니다. 이러한 향상 된 인증 기능을 통해 사용자는 Facebook, Twitter, Windows Live, Google 등의 계정에서 기존 자격 증명을 사용 하 여 사이트에 로그인 할 수 있습니다. 기본적으로 템플릿은 SQL Server Express LocalDB의 인스턴스에 기본 데이터베이스 이름을 사용 하 여 멤버 자격 데이터베이스를 만듭니다. 여기에는 웹 Visual Studio Express 2013와 함께 제공 되는 개발 데이터베이스 서버가 있습니다.
- **클라이언트 스크립트 및 클라이언트 프레임 워크**-ASP.NET Web Form 페이지에 클라이언트 스크립트 기능을 포함 하 여 ASP.NET의 서버 기반 기능을 향상 시킬 수 있습니다. 클라이언트 스크립트를 사용 하 여 보다 풍부 하 고 응답성이 뛰어난 사용자 인터페이스를 사용자에 게 제공할 수 있습니다. 또한 브라우저에서 페이지가 실행 되는 동안 클라이언트 스크립트를 사용 하 여 웹 서버에 대 한 비동기 호출을 수행할 수 있습니다.
- **라우팅**URL 라우팅을 사용 하면 실제 파일에 매핑되지 않는 요청 url을 허용 하도록 응용 프로그램을 구성할 수 있습니다. 요청 URL은 웹 사이트에서 페이지를 찾기 위해 사용자가 브라우저에 입력 하는 URL입니다. 라우팅을 사용 하 여 사용자에 게 의미 있는 의미 있는 Url을 정의 하 고 SEO (검색 엔진 최적화)에 도움이 될 수 있습니다.
- **상태 관리**-ASP.NET Web Forms에는 페이지 단위와 응용 프로그램 전체의 데이터를 유지 하는 데 도움이 되는 몇 가지 옵션이 포함 되어 있습니다.
- **보안**-보다 안전한 응용 프로그램을 개발 하는 중요 한 부분은 it에 대 한 위협을 이해 하는 것입니다. Microsoft는 스푸핑, 변조, 부인, 정보 공개, 서비스 거부, 권한 상승 (STRIDE) 등의 위협을 분류 하는 방법을 개발 했습니다. ASP.NET Web Forms에서 ASP.NET Web Forms의 다양 한 보안 동작을 사용자 지정할 수 있는 확장성 지점과 구성 옵션을 추가할 수 있습니다.
- **성능**-성공적인 웹 사이트 또는 프로젝트에서 핵심 요소가 될 수 있습니다. ASP.NET Web Forms를 사용 하면 페이지 및 서버 제어 처리, 상태 관리, 데이터 액세스, 응용 프로그램 구성 및 로드, 효율적인 코딩 방법과 관련 된 성능을 수정할 수 있습니다.
- ASP.NET Web Forms를 사용 하면 브라우저의 기본 설정 언어 설정이 나 사용자의 명시적 선택 언어에 따라 콘텐츠와 기타 데이터를 가져올 수 있는 웹 페이지를 만들 수 있습니다. 콘텐츠 및 기타 데이터는 리소스 라고 하며 이러한 데이터는 리소스 파일 또는 다른 원본에 저장할 수 있습니다. ASP.NET Web Forms 페이지에서 컨트롤을 구성 하 여 리소스에서 해당 속성 값을 가져옵니다. 런타임에 리소스 식은 적절 한 지역화 된 리소스 파일의 리소스로 대체 됩니다.
- **디버깅 및 오류 처리**-ASP.NET에는 Web Forms 응용 프로그램에서 발생할 수 있는 문제를 진단 하는 데 도움이 되는 기능이 포함 되어 있습니다. ASP.NET Web Forms 내에서 디버깅 및 오류 처리는 잘 지원 되므로 응용 프로그램을 효과적으로 컴파일하고 실행할 수 있습니다.
- **배포 및 호스팅**-Visual Studio, ASP.NET, AZURE 및 IIS는 Web Forms 응용 프로그램을 배포 하 고 호스팅하는 프로세스에 도움이 되는 도구를 제공 합니다.

## <a name="deciding-when-to-create-a-web-forms-application"></a>Web Forms 응용 프로그램을 만들어야 하는 경우 결정

ASP.NET Web Forms 모델 또는 ASP.NET MVC 프레임 워크와 같은 다른 모델을 사용 하 여 웹 응용 프로그램을 구현할지 여부를 신중 하 게 고려해 야 합니다. MVC 프레임워크가 Web Forms 모델을 대체하지는 않지만, 두 가지 프레임워크 모두 웹 응용 프로그램에 사용할 수 있습니다. 특정 웹 사이트에 대 한 Web Forms 모델 또는 MVC 프레임 워크를 사용 하도록 결정 하기 전에 각 방법의 장점을 비교 합니다.

### <a name="advantages-of-a-web-forms-based-web-application"></a>Web Forms 기반 웹 응용 프로그램의 이점

Web Forms 기반 프레임워크에서 제공하는 이점은 다음과 같습니다.

- HTTP상의 상태를 보존하는 이벤트 모델을 지원하므로 LOB(Line-Of-Business) 웹 응용 프로그램 개발에 도움이 됩니다. Web Forms 기반 응용 프로그램에서는 수백 개의 서버 컨트롤에서 지원하는 수십 가지 이벤트를 제공합니다.
- 또한 개별 페이지에 기능을 추가하는 페이지 컨트롤러 패턴을 사용합니다. 자세한 내용은 MSDN 웹 사이트의 [페이지 컨트롤러](https://go.microsoft.com/fwlink/?LinkId=106359 "페이지 컨트롤러") 를 참조 하십시오.
- 상태 정보를 더 쉽게 관리할 수 있는 뷰 상태나 서버 기반 양식을 사용 합니다.
- 적은 수의 웹 개발자로 구성된 팀과 신속한 응용 프로그램 개발에 사용 가능한 다수의 구성 요소를 활용하려는 디자이너에게 매우 적합합니다.
- 일반적으로 구성 요소 ( **페이지** 클래스, 컨트롤 등)가 긴밀 하 게 통합 되어 있으며 일반적으로 MVC 모델 보다 작은 코드가 필요 하기 때문에 응용 프로그램 개발에는 더 복잡 하지 않습니다.

### <a name="advantages-of-an-mvc-based-web-application"></a>MVC 기반 웹 응용 프로그램의 이점

ASP.NET MVC 프레임워크에서 제공하는 이점은 다음과 같습니다.

- 응용 프로그램을 모델, 뷰 및 컨트롤러로 분할해 복잡성을 관리하기 쉽습니다.
- 뷰 상태 또는 서버 기반 양식을 사용하지 않습니다. 따라서 MVC 프레임워크는 응용 프로그램의 동작을 완벽히 제어하려는 개발자에게 적합합니다.
- 단일 컨트롤러를 통해 웹 응용 프로그램 요청을 처리하는 프런트 컨트롤러 패턴을 사용합니다. 따라서 풍부한 라우팅 인프라를 지원하는 응용 프로그램을 디자인할 수 있습니다. 자세한 내용은 MSDN 웹 사이트의 [프런트 컨트롤러](https://go.microsoft.com/fwlink/?LinkId=106357 "프런트 컨트롤러") 를 참조 하십시오.
- TDD(테스트 구동 방식 개발)에 대해 향상된 지원을 제공합니다.
- 응용 프로그램 동작을 매우 많이 제어 해야 하는 개발자 및 웹 디자이너의 대규모 팀에서 지 원하는 웹 응용 프로그램에 적합 합니다.
