<span data-ttu-id="7df95-243">런타임 오류는 사이트를 방문 하는 사용자 로부터 잠재적으로 중요 한 오류 정보를 숨기는 반면, 사이트의 모양과 느낌에서 중단 되 고 응용 프로그램을 버그가 있는 합니다.</span><span class="sxs-lookup"><span data-stu-id="7df95-243">While the Runtime Error YSOD hides potentially sensitive error information from the user visiting the site, it breaks from your site's look and feel and makes your application look buggy.</span></span> <span data-ttu-id="7df95-244">사용자 지정 오류 페이지를 만들고 디자인 하 고 `<customErrors>` 섹션의 `defaultRedirect` 특성에 해당 URL을 지정 하는 사용자 지정 오류 페이지를 사용 하는 것이 더 나은 방법입니다.</span><span class="sxs-lookup"><span data-stu-id="7df95-244">A better approach is to use a custom error page, which entails creating and designing the custom error page and specifying its URL in the `<customErrors>` section's `defaultRedirect` attribute.</span></span> <span data-ttu-id="7df95-245">여러 HTTP 오류 상태에 대해 여러 사용자 지정 오류 페이지를 포함할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="7df95-245">You can even have multiple custom error pages for different HTTP error statuses.</span></span>

<span data-ttu-id="7df95-246">사용자 지정 오류 페이지는 프로덕션 환경에서 웹 사이트에 대 한 포괄적인 오류 처리 전략의 첫 번째 단계입니다.</span><span class="sxs-lookup"><span data-stu-id="7df95-246">The custom error page is the first step in a comprehensive error handling strategy for a website in production.</span></span> <span data-ttu-id="7df95-247">오류를 개발자에 게 경고 하 고 세부 정보를 기록 하는 것도 중요 한 단계입니다.</span><span class="sxs-lookup"><span data-stu-id="7df95-247">Alerting the developer of the error and logging its details are also important steps.</span></span> <span data-ttu-id="7df95-248">다음 세 가지 자습서에서는 오류 알림 및 로깅에 대 한 기술을 살펴봅니다.</span><span class="sxs-lookup"><span data-stu-id="7df95-248">The next three tutorials explore techniques for error notification and logging.</span></span>
